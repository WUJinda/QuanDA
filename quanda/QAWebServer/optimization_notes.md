# K线数据加载性能优化说明

## 优化内容

### 1. 后端优化

#### 数据查询优化
- 添加 `limit` 参数，限制返回数据量（分钟线默认2000条）
- MongoDB查询使用倒序+limit，避免全表扫描
- 只返回必要的字段，减少数据传输量

#### 缓存优化
- 添加HTTP缓存头 `Cache-Control`
- 日线数据缓存5分钟
- 分钟数据缓存1分钟

### 2. 前端优化

#### 数据缓存
- 实现5分钟内存缓存机制
- 相同请求直接从缓存返回
- 减少重复的网络请求

#### 计算优化
- 布林带计算使用滑动窗口算法，时间复杂度从 O(n²) 降至 O(n)
- 避免重复计算均值和方差

#### 渲染优化
- 数据量超过5000条时自动抽样显示
- 关闭ECharts动画效果
- 启用大数据量优化模式（large: true）
- 使用LTTB算法进行线条抽样
- 使用notMerge模式更新图表

### 3. 建议的MongoDB索引

```javascript
// 在MongoDB中创建索引以提升查询性能
db.future_day.createIndex({ "code": 1, "date": -1 })
db.future_min.createIndex({ "code": 1, "datetime": -1 })
```

## 性能提升预期

- 数据查询速度：提升 50-70%
- 前端渲染速度：提升 60-80%
- 重复请求响应：提升 90%+（缓存命中）
- 大数据量场景：提升 70-90%

## 使用建议

1. 分钟线数据建议限制在2000条以内
2. 日线数据可以不限制或限制在5000条
3. 如需查看更多历史数据，建议分批加载
4. 定期清理过期的缓存数据
